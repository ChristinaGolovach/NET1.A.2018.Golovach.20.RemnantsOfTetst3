Для решения данной задачи использовался паттерн Visitor, чтобы в дальнейшем можно было с легкостью добавлять новую функциональность, не изменя при это API существующих типов. Для реализации паттерна использовался dynamic, т.к предполагаемые новые типы могут разрабатываться другим человеком, и он может не заложить логику метода Acept().
